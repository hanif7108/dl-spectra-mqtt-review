Overview

Seven headline numbers establishing the design envelope. Click a card for a short explanation.

Onsite EEWS face three coupled failure modes: (i) a near-field blind zone of ~38 km, (ii) saturation of canonical magnitude proxies above M_w 7, and (iii) dependence on catalog hypocenters that arrive 30–60 s after P-onset. This project introduces a hybrid MQTT-based inter-service architecture that elevates the messaging substrate from an implementation detail to a first-class design contribution. Eight containerised services exchange raw waveforms, engineered features, and retained alerts through an MQTT v5 broker with a formally specified topic tree, QoS differentiation, and ACL model.

The architecture reuses the upstream IDA-PTW deep-learning pipeline (R² = 0.729 over 25 058 accelerograms) and adds four messaging-enabled refinement mechanisms. A closed-form latency budget establishes ≤ 780 ms of overhead plus the P-wave observation window, delivering sub-second near-field binary alerts at t ≈ 630 ms from P-onset.

Why This Matters

Four structural problems that the existing onsite EEWS paradigm cannot fully resolve on its own. Each card summarises one problem in under ten lines.

1

Near-field blind zone

The P–S travel-time difference within ~38 km is insufficient for a 3 s observation window plus 2 s dissemination — the events most lethal to dense cities like Bandung and Jakarta sit inside this gap.

2

Parameter saturation

τ_c and P_d saturate above M_w 7: the 3 s window sees only the rupture nucleation phase. The 2011 Tohoku-Oki event settled this debate — its initial P coda showed no feature predicting the M_w 9.0 outcome.

3

Catalog dependence

Classical onsite regressors need a catalog hypocenter (latency 30–60 s). IDA-PTW Stage 1.5 removes this by auto-regressing distance from the P-wave at 99.87 % fidelity — but the catalog-free guarantee must be preserved end-to-end.

4

Subscriber bandwidth

A 103-period Sa(T) vector is overkill for a building whose fundamental period is a single value. Naïvely fanning the full vector to 50 000 buildings in Jakarta wastes bandwidth and pushes correction logic onto every edge agent.

Fusion & Refinement Mechanisms

Four mechanisms enabled by the messaging substrate that raise accuracy beyond single-station onsite inference. Green cards are the highest priority — they attack the dominant φ variance component.

M1

Multi-station Bayesian fusion

Inverse-variance weighted mean of log10 Sa(T) across N co-triggering stations.

σ → 0.55 for N = 3
M2

Progressive alert revision

Publish at t_P + 3, 5, 8 s with the same event_id and monotonically increasing revision.

+0.5% R² per additional second
M3

Edge site projection

ADR-0005 projector applies Δ_Vs30 + Δ_dist + Δ_HV per site and emits a single Sa(T₁) per event.

attacks φ directly
M4

Ensemble hypocenter

Grid-search localisation from ≥ 3 Stage-1.5 distance regressor outputs sharpens log10(dist) input.

+1–2% R² expected
M5

Spectrogram 2D branch

Parallel multi-scale STFT CNN fused linearly with the 1-D DLUHS2 backbone (Münchmeyer 2025 precedent).

+3–5% R²
M6

Physics-guided loss

Penalise predictions that exceed Abrahamson 2014 GMPE ± 2σ; use SHAP for feature re-weighting.

tighter p16/p84

Subscriber Contract Per Site

Three subscription patterns. Recommended default for operational sites is per-site projection (ADR-0005).

Near-field only

eews/v1/pred/urpd/IA/CMJI

Minimum payload for life-safety shutdown in sub-second budget.

  • p_prob, near_field_flag
  • spectral_centroid_hz
  • Latency budget ≈ 630 ms

Full-spectrum analyst

eews/v1/alert/jawa_barat/+

Whole 103-period vector with fused uncertainty bounds — for regional dashboards and SCADA.

  • 103-period log10 Sa(T)
  • p16 / p84 bounds (M1)
  • revision, ptw_s (M2)
  • SNI-1726:2019 category

Operational building RECOMMENDED

eews/v1/alert_site/UI-FT-B

Single Sa(T₁) number per event revision, already site-corrected. Minimum bandwidth at the edge.
